What is HTML?

HyperText Markup Language (HTML) is the skeleton of every webpage. It defines the meaning and structure of content using a system of elements represented by tags. A tag is written between angle brackets — <tagname>.

The Document Skeleton

HTML<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My First Page</title> </head> <body> <h1>Hello World!</h1> <p>My <b>first</b> webpage.</p> </body> </html>
💡 Key Insight

Every HTML document needs <!DOCTYPE html> at the top — it tells the browser to use modern HTML5 standards. Without it, browsers enter "quirks mode" and may render things unexpectedly.

Heading Hierarchy

HTML provides six heading levels (h1 through h6). h1 is the most important and should be used once per page, like a book's title. Always use headings in order — never skip from h1 to h4.

Images & Accessibility

The <img> tag is self-closing — it has no closing tag. The alt attribute is essential: screen readers use it to describe the image to visually impaired users, and search engines use it for indexing.

HTML<!-- Good practice --> <img src="profile.jpg" alt="Nitin smiling at camera" width="400" height="300" loading="lazy" >

Tables — Structured Data

HTML tables use a row-first model: <tr> creates a row, then <td> fills cells within it. Use colspan and rowspan to merge cells across columns or rows — great for complex data layouts.

⚠️ Common Mistake

Never use tables for page layout — that's what CSS is for. Tables are strictly for tabular data (schedules, comparisons, spreadsheets). Misusing tables breaks accessibility and responsiveness.

Forms — How the Web Listens

The <form> element wraps all inputs and has two critical attributes: action (where data is sent) and method (GET appends data to the URL; POST sends it hidden in the request body — always use POST for sensitive data).

HTML<form action="/submit" method="POST"> <label for="email">Email</label> <input type="email" id="email" name="email" placeholder="you@example.com" required > <button type="submit">Submit</button> </form>
💡 Best Practice

Always link <label> to its input using matching for and id attributes. This expands the clickable area and greatly improves accessibility for keyboard and screen-reader users.

Why Semantic HTML Matters

Using <header>, <main>, <nav>, and <footer> instead of generic <div> tags tells search engines and assistive technologies what each section of the page means — massively improving SEO and accessibility.

CSS Selector Power

CSS/* Element selector */ p { color: #607080; } /* Class selector */ .card { background: #111; } /* Pseudo-class */ a:hover { color: #00e5ff; } /* Combinator — direct child */ nav > a { font-weight: 600; } /* Attribute selector */ input[type="email"] { border: 1px solid cyan; }
📌 CSS Specificity

Specificity determines which rule wins when multiple rules target the same element. IDs (100) beat classes (10), which beat elements (1). !important overrides all — but use it sparingly as a last resort.

The Box Model — Everything is a Box

Every HTML element is a rectangular box. From outside to inside: margin (space outside) → border → padding (space inside) → content. Setting box-sizing: border-box makes width calculations intuitive — padding and border are included inside the declared width.

Flexbox — One Axis at a Time

CSS.container { display: flex; flex-direction: row; /* or column */ justify-content: space-between; /* main axis */ align-items: center; /* cross axis */ gap: 1.5rem; /* space between */ flex-wrap: wrap; /* allow wrapping */ }
🧠 Mental Model

Think of Flexbox as one-dimensional layout — it handles either a row or a column at a time. For two-dimensional grid layouts (rows AND columns simultaneously), reach for CSS Grid instead.

Variables — Choosing the Right Keyword

Prefer const by default. Use let when you need to reassign. Avoid var — it has function scope and hoisting behaviour that causes subtle bugs. Always declare before use.

DOM Manipulation

JavaScript// Select an element const btn = document.querySelector('#myBtn'); // Listen for a click btn.addEventListener('click', () => { const box = document.getElementById('box'); box.style.background = '#00e5ff'; box.textContent = 'Clicked! ✓'; }); // Loop example for (let i = 0; i < 5; i++) { console.log(`Item ${i}`); }
🔑 Core Concept

The DOM (Document Object Model) is a live tree of objects representing your HTML. JavaScript can read and rewrite any part of this tree in real time — that's what makes web pages dynamic and interactive.

localStorage — Persistence Without a Server

localStorage is a browser API that stores up to ~5MB of key-value string data permanently — it survives page refreshes and browser restarts, unlike session variables.

JavaScript// Save tasks array const saveTasks = (tasks) => { localStorage.setItem('tasks', JSON.stringify(tasks)); }; // Load on page open const tasks = JSON.parse( localStorage.getItem('tasks') || '[]' ); // Mark complete task.done = !task.done; saveTasks(tasks);
